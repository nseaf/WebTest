# AI-Agent Web渗透测试系统

## 项目概述

本项目是一个基于 Claude Code 的多Agent Web渗透测试系统。通过层级式多Agent架构，利用AI技术模拟人工前端Web测试，实现自主探索、安全测试和漏洞发现。

### Agent职责

| Agent | 职责 |
|-------|------|
| **Coordinator Agent** | 任务规划、事件队列管理、人机交互代理、并行调度、流程审批协调 |
| **AccountParser Agent** | 权限文档解析、账号配置生成、接口文档解析、测试Checklist生成 |
| **Navigator Agent** | 页面导航、链接跟踪、多Chrome实例管理、会话状态监控 |
| **Scout Agent** | 页面分析、元素识别、链接发现、API发现、敏感数据检测 |
| **Form Agent** | 表单识别、智能填写、登录执行、验证码检测、审批操作记录 |
| **Security Agent** | 越权测试(IDOR)、注入测试、流程审批越权测试、请求重放 |
| **Analyzer Agent** | 重放结果分析、漏洞判定、探索建议生成 |

## 技术栈

| 组件 | 技术 | 说明 |
|------|------|------|
| **Agent框架** | Claude Code | 基于Prompt的角色扮演，非注册Agent |
| **浏览器自动化** | browser-use CLI + Skill | 主要工具，支持多Chrome实例管理 |
| **浏览器自动化(备用)** | Playwright MCP | 特殊场景使用，更灵活的API |
| **安全测试** | BurpBridge MCP | BurpSuite插件，提供历史记录和请求重放 |
| **数据存储** | MongoDB | BurpBridge依赖，存储历史请求和重放结果 |

## 关键特性

### 多Chrome实例管理
- 每个账号独立的Chrome实例和用户数据目录
- 独立的CDP端口和browser-use session
- 成对创建和关闭机制

### 登录态保持
- Cookie管理和自动重新登录
- 验证码检测和人机交互
- 会话过期检测和处理

### API发现
- 分析网络请求，发现隐藏API
- API模式识别（如 `/api/users/{id}`）
- 敏感数据检测

### 并行架构
- Security Agent与探索Agent并行运行
- 事件队列驱动的Agent通信
- 实时漏洞发现

### 流程审批测试
- 权限文档智能解析（Excel/CSV/Markdown）
- 多账号按顺序审批调度
- 请求重放越权测试（不影响原流程）
- 测试矩阵自动生成

## 快速开始

### 1. 环境准备

```bash
# 启动 MongoDB
docker run -d --name mongodb -p 27017:27017 mongo:latest

# 构建 BurpBridge 插件
cd /path/to/BurpBridge && mvn clean package

# 在 Burp Suite 中加载插件
# 加载 target/BurpBridge-1.0-SNAPSHOT.jar

# 安装 browser-use CLI
pip install browser-use

# 安装 Playwright 浏览器
npx playwright install
```

### 2. 配置MCP服务

在 `.mcp.json` 中配置 Playwright 代理：

```json
{
  "playwright": {
    "command": "npx",
    "args": [
      "@playwright/mcp@latest",
      "--proxy-server", "http://127.0.0.1:8080"
    ]
  }
}
```

### 3. 启动测试

```
你现在扮演Coordinator Agent角色。
请阅读 agents/coordinator.md 了解你的职责。
目标URL: https://www.baidu.com
请开始规划并执行Web探索测试。
```

### 4. 流程审批测试

```
你现在扮演AccountParser Agent角色。
请阅读 agents/account_parser.md 了解你的职责。
权限文档路径: C:\Users\xxx\权限文档.xlsx
请解析权限文档并生成配置。
```

## 流程审批测试指南

### 测试策略

流程审批场景采用**请求重放测试**策略：
- 不实际执行审批操作
- 捕获请求后用其他角色的Cookie重放
- 分析响应判断是否存在越权漏洞
- 不影响原流程状态

### 测试流程

```
1. 解析权限文档
   AccountParser Agent → config/accounts.json + result/workflow_config.json

2. 创建多账号实例
   每个角色独立的Chrome实例 → 不同CDP端口

3. 按顺序执行审批
   Form Agent执行审批 → Scout Agent发现API → Security Agent越权测试

4. 生成测试报告
   测试矩阵 + 漏洞列表
```

| 文件 | 用途 |
|------|------|
| `agents/account_parser.md` | 权限文档解析规则 |
| `result/workflow_config.json` | 流程审批节点配置 |
| `result/workflow_test_matrix.json` | 越权测试矩阵 |

## 故障排查

### Burp同步记录为空

1. 确保 Burp Suite Intercept 模式已关闭
2. 检查 Playwright 是否配置了 `--proxy-server` 参数
3. 运行 `curl http://localhost:8090/health` 确认 BurpBridge 正常
4. 确认 MongoDB 容器运行中

### MCP连接失败

1. 运行 `/mcp` 查看 MCP 服务状态
2. 检查 `.mcp.json` 配置格式
3. 重启 Claude Code 会话重新加载配置
